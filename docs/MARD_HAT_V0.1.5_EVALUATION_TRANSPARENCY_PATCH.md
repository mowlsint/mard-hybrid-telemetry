# MARD-HAT v0.1.5 Evaluation / Transparency Patch

## Purpose

This patch makes MARD-HAT more suitable for public evaluation and maritime use:

1. clearer score semantics in `public/index.html`;
2. more transparent README/readme wording;
3. maritime relevance filtering for FIMI-lite;
4. cautious obscuration / information-fog explanation;
5. useful downloads for own analysis;
6. source-health states that distinguish `ok`, `partial`, `error` and `disabled`.

## Main changes

### Score

The dashboard now presents the HAT score as a 0–100% contextual pressure score with explicit bands:

```text
0–20   green        normal
21–40  yellow       elevated
41–60  orange       watch
61–80  deep orange  high
81–100 red          critical
```

This is not a probability and not attribution-grade.

### Source health

Source health now supports partial states. A source with at least one successful query and one failed/rate-limited query becomes `partial`, not `ok`.

Source coverage uses weighted source health:

```text
ok       = 1.0
partial  = 0.5
degraded = 0.35
error    = 0.0
disabled = 0.0
```

### Maritime FIMI-lite

GDELT items are post-filtered. An article becomes score-relevant only if it has:

```text
maritime anchor + threat/FIMI anchor
```

For Core MARD-Eu it also needs a North/Baltic/Northern-Europe anchor.

Global maritime spillover such as Hormuz, Iran, Suez, Red Sea or oil blockade narratives remains visible but down-weighted.

### Obscuration context

A new `obscuration_context` object explains when the telemetry is compatible with information fog / obscuration. It explicitly avoids claiming intentional hiding, attribution or proof.

### Downloads

Build output now writes:

```text
public/downloads/hat_analyst_bundle_latest.json
public/downloads/fimi_maritime_articles_latest.csv
public/downloads/fimi_maritime_articles_latest.json
public/downloads/source_health_latest.csv
public/downloads/README_downloads.txt
```

These files are intended for review, charts, notebooks and customer extracts.
